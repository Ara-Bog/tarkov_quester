# Квестовик — планировщик рейда EFT

Веб-приложение для отслеживания квестов Escape from Tarkov и планирования рейдов
«что взять с собой» в разрезе локаций. Без бэкенда: данные — статический JSON,
прогресс хранится в браузере (IndexedDB).

## Страницы

- **Главная** (по клику на логотип «Квестовик») — что это, зачем, контакты, источник данных.
- **Мои квесты** — только активные квесты (которые вы взяли). При выполнении квеста
  следующие по цепочке добавляются сюда автоматически. Сверху — **быстрая настройка**:
  если вы в середине игры, найдите свой текущий квест (напр. «Оружейник. Часть 6») —
  все предыдущие по ветке отметятся «Выполнен». Там же кнопка **сброса всего прогресса**.
- **Все квесты** (510+ из [tarkov.dev](https://tarkov.dev), на русском) — каталог; кнопка
  **«Добавить в мои квесты»** появляется в раскрытии квеста. По умолчанию уже взятые квесты
  скрыты (чекбокс «Показывать мои квесты»). Колонки сортируются (клик по заголовку). Для каждого квеста видно,
  что он *требует*, что *открывает*, какие квесты *провалятся при его выполнении*. Статус
  (Доступен/Заблокирован/Выполнен/Провален) и авто-провал считаются автоматически.
- **План рейда** — под-вкладки:
  - **Мой план** — отмеченные галочкой ⚑ квесты, собранные по локациям: **взять с собой /
    заложить** и **сдать** (отдельно «купить / найти» и «только из рейда (FiR)»). Справа под
    итогом — блок **«Список находок»** с пометкой группы у каждого предмета.
  - **по локации** (вкладка есть для каждой карты, даже без активных квестов; на «Мой план» есть
    ещё выбор карты + кнопка «Показать карту») — **карта локации** с геопозиционированными
    маркерами: цели текущих квестов (цветная точка на квест), **выходы** (треугольник, по
    фракциям), **зоны спавна боссов** (красный ромб) + список боссов с шансами. Слои
    включаются/выключаются. У многоэтажных карт — **переключатель этажей** (показывает слой SVG
    нужного уровня: Земля / Подвал / 2 этаж …). Доступны все карты с векторной графикой, включая
    Лабораторию и Терминал (Лабиринт/Ледокол у tarkov.dev только тайлами — их нет).
    Масштаб от «вся карта целиком» до приближения (карта встроена как **inline-SVG** и остаётся
    векторно-чёткой на любом зуме), панорама перетаскиванием, **режим во весь экран** (⛶) с угловой
    панелью текущих задач. При отметке выполнения маркер скрывается. Плюс ссылка на интерактивную
    карту tarkov.dev.
  Чекбокс в шапке таблицы добавляет все показанные квесты в план сразу (с предупреждением, если их >5).

  > Карты и координаты взяты из открытых данных tarkov.dev: SVG-карты с `assets.tarkov.dev`,
  > геопозиции целей/выходов/боссов — из API (`TaskZone.position`, `MapExtract.position`,
  > `MapSpawn` с категорией boss), а формула размещения маркеров (`build/maps.mjs`, `geoFrac` в
  > `app.js`) портирована из их `getCRS()` (репозиторий `the-hideout/tarkov-dev`). Маркеры целей
  > есть для ~570 задач; для карт без конфигурации (Лаборатория, Лабиринт, Ледокол) — список
  > задач + ссылка на карту.
- **Список находок** — личные списки предметов по группам с описанием (напр. «На увеличение
  схрона»), с **искомым количеством** и отметкой «найдено». Поиск по полному каталогу (~5000 предметов).
- **Каппа** — отслеживание предметов для контейнера Каппа (квест «Коллекционер»): чек-лист с
  прогресс-баром, поиском и скрытием собранного.

**Прогресс по целям.** У каждой цели квеста (в раскрытии и в плане рейда) есть отметка
выполнения: чекбокс или счётчик «сделано N из M». Это уменьшает, что нужно взять/сдать в плане
(напр. поставил 2 камеры из 4 — в плане останется 2), и показывает остаток по сдаваемым
предметам (сдал 3 из 5 жетонов — нужно ещё 2). Выполненные цели в плане **зачёркиваются**,
а не пропадают — отметку можно снять. У каждой цели есть ссылка **«где? ↗»** на гайд по квесту.

При **первом открытии** сайта показывается **Главная** и плашка про куки (используется ровно
одна кука `th_visited`, чтобы не показывать главную/плашку повторно), далее — «Мои квесты».
Текущая вкладка/карта сохраняется в URL-хэше — обновление страницы не сбрасывает её.

Фильтры (квесты): поиск, локация, торговец, действие, статус, фракция, «только Каппа»,
«только в плане», «показать выполненные» (по умолчанию выполненные скрыты). Клик по иконке
предмета открывает её в полном размере.

**Хранение** прогресса, плана и списков находок — в IndexedDB (фолбэк на localStorage / память).
Есть версионирование с миграциями (`DB_VERSION` — структура хранилищ, `DATA_VERSION` — формат
данных в `app.js`): при обновлении сайта данные существующих пользователей мигрируются, а не ломаются.
Колонки в таблицах сортируются по клику. Бóльшие иконки в столбцах «Взять с собой»/«Сдать».

## Запуск

Откройте `index.html`. Двумя способами:

1. **Двойной клик** по `index.html` (file://) — работает за счёт `data/quests-data.js`
   (тот же JSON, что и `quests.json`, но как глобальная переменная; обходит блокировку
   `fetch` для локальных файлов).
2. **Через локальный сервер** (рекомендуется) — тогда грузится `data/quests.json`:
   ```
   python -m http.server 8777
   # затем открыть http://127.0.0.1:8777/
   ```

Никакой сборки для запуска не нужно — это статические файлы.

## Размещение на GitHub Pages

Проект — чистая статика, разворачивается на GitHub Pages из корня репозитория:
1. Залить файлы в репозиторий.
2. Settings → Pages → Source: `Deploy from a branch`, ветка `main`, папка `/ (root)`.

В репозитории есть `.nojekyll` (чтобы Pages отдавал файлы как есть) и `.gitignore`
(исключает регенерируемый `build/tasks_raw.json` и временные файлы). Все пути относительные,
поэтому сайт работает и в подпапке `username.github.io/<repo>/`.

## Структура

```
index.html            — разметка
styles.css            — стили (тёмная тема)
app.js                — вся логика (хранилище, статусы, фильтры, планировщик, находки)
data/quests.json      — данные квестов (этот файл можно передавать другим)
data/quests-data.js   — те же данные как window.__QUESTS__ (фолбэк для file://)
data/items.json       — полный каталог предметов для «Списка находок» (~5000)
data/items-data.js    — тот же каталог как window.__ITEMS__ (фолбэк для file://)
build/fetch.mjs       — загрузка квестов с tarkov.dev -> build/tasks_raw.json
build/transform.mjs   — преобразование -> data/quests.json (+ .js)
build/items.mjs       — загрузка каталога предметов -> data/items.json (+ .js)
build/maps.mjs        — гео-конфиг карт (SVG + координаты) -> data/maps-geo.json (+ .js)
```

## Регенерация данных

Нужен Node.js 18+ (используется нативный `fetch`).

```
node build/fetch.mjs       # скачать свежие квесты с tarkov.dev (с координатами целей)
node build/transform.mjs   # пересобрать data/quests.json и quests-data.js
node build/items.mjs       # пересобрать каталог предметов data/items.json
node build/maps.mjs        # пересобрать гео-конфиг карт data/maps-geo.json
```

Схема `quests.json` описана в [`data/SCHEMA.md`](data/SCHEMA.md).

Данные предоставлены [tarkov.dev](https://tarkov.dev) (open-source, под лицензией контента BSG).
