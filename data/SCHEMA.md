# Схема `quests.json`

Сгенерировано `build/transform.mjs` из API tarkov.dev. Один объект:

```jsonc
{
  "meta":   { "source", "lang", "taskCount", "itemCount", "note" },
  "maps":   [ { "id": "map_1", "name": "Эпицентр" }, ... ],   // канонические локации (ночь/21+ слиты)
  "traders":[ { "id": "...",   "name": "Прапор" }, ... ],
  "items":  { "<itemId>": { "id", "name", "shortName", "icon", "img", "quest"? } },  // icon=маленькая, img=512px
  "tasks":  [ Task, ... ]
}
```

## Task

```jsonc
{
  "id": "657...",                  // id квеста (tarkov.dev)
  "name": "Что-то мне это напоминает...",
  "normalizedName": "...",
  "trader": "<traderId>",          // ссылка в meta.traders
  "faction": null | "USEC" | "BEAR",
  "minLevel": 10 | null,
  "prestige": 1,                   // (опц.) требуемый уровень престижа; есть у версий «Новое начало»
  "kappa": true,                   // нужен для контейнера Каппа
  "lightkeeper": false,            // нужен для Смотрителя
  "restartable": false,
  "wiki": "https://...",
  "image": "https://...",
  "maps": ["map_3", ...],          // объединение локаций всех целей
  "requires": [                    // предусловия (граф разблокировки)
    { "task": "<taskId>", "status": ["complete"] }   // status: complete | active | failed
  ],
  "failedBy": ["<taskId>", ...],   // этот квест ПРОВАЛИТСЯ, если выполнить любой из перечисленных
  "objectives": [ Objective, ... ]
}
```

## Objective (цель)

```jsonc
{
  "id": "...",
  "type": "plantItem",             // исходный тип tarkov.dev (giveItem, mark, shoot, visit, extract, ...)
  "action": "Камера",              // человекочитаемое действие (RU): Найти/Сдать/Оставить/Маркер/Камера/Убить/Посетить/Выход/...
  "description": "Установить WI-FI Камеру ...",
  "optional": false,
  "maps": ["map_3"],               // на каких локациях выполняется
  "count": 1 | null,
  "target": "Выход Такси" | null,  // цель убийства / название выхода
  "bring": [                       // ВЗЯТЬ С СОБОЙ / ЗАЛОЖИТЬ (маркеры, камеры, оружие, заложить предмет)
    { "item": "<itemId>", "count": 2, "kind": "marker" }   // kind: plant|camera|marker|use|weapon
    // или выбор из нескольких:  { "anyOf": ["<id>", "<id>"], "count": 1, "kind": "weapon" }
  ],
  "handIn": [                      // НАЙТИ/КУПИТЬ И СДАТЬ торговцу
    { "item": "<itemId>", "count": 3, "fir": true, "kind": "handin" },   // fir=found-in-raid (нельзя купить)
    { "anyOf": ["<id>"], "count": 1, "fir": false, "kind": "handin" },   // один из нескольких
    { "category": "Передать любые предметы медицины…", "count": 3, "fir": true,
      "items": ["<id>", "<id>", ...], "kind": "category" }, // подходит любой из items этой категории
    { "item": "<weaponId>", "count": 1, "kind": "build" }                // собрать оружие на верстаке и сдать
  ],
  "keys": [                        // ключи, которые нужно взять (это тоже "взять с собой")
    "<itemId>",                    // один ключ
    { "anyOf": ["<id>", "<id>"] }  // один из нескольких
  ]
}
```

### Логика статуса (как считает приложение)

- `completed` — отмечен пользователем.
- `failed` — отмечен пользователем **или** авто-провал: выполнен какой-то квест из его `failedBy`.
- `available` — не выполнен/не провален и все `requires` удовлетворены по статусу:
  - `complete` → нужный квест **выполнен**;
  - `active` → нужный квест **взят/в работе** (в «Моих квестах») или уже выполнен — завершать его
    не обязательно (это альтернативные ветки);
  - `failed` → нужный квест **провален**.
- `locked` — иначе.

> **Ветвление.** Завершение квеста, который указан в чужом `failConditions`, проваливает альтернативу
> (`failedBy`), а зависящие от её провала квесты (`requires` со статусом `failed`) становятся доступны
> и авто-добавляются в «Мои квесты». Пример: «Реагент. Часть 4» (complete) → «Большой заказчик»/«Простое
> любопытство» (failed) → «Без обид»/«Забудем старые обиды»/«Безопасный коридор» (available).

### Планировщик рейда

- **Взять с собой / заложить** — суммируются `bring` + `keys` целей выбранных квестов,
  сгруппированные по локации (`maps[0]`). Ключи = 1 шт. независимо от числа целей.
- **Найти / купить и сдать** — суммируются `handIn`. Пара `findItem`+`giveItem` одного
  предмета в одном квесте считается один раз (по максимуму), категории и сборки — отдельно.
- Группы `anyOf` («одно из») дают выбор в интерфейсе.

## `items.json` — полный каталог предметов

Отдельный файл для страницы «Список находок» (личные списки предметов).
`{ meta, items: [ { id, name, shortName, icon, img, cat } ] }` — ~5000 предметов.
Генерируется `node build/items.mjs`.

## `story-quests.json` — сюжетные квесты (главы историй)

Этих квестов нет в API tarkov.dev — файл **ведётся вручную** (с русской вики), без build-скрипта.
Ключевое отличие от обычных квестов: подзадачи сгруппированы в **стадии**, открывающиеся постепенно,
и поддержано **ветвление** (несколько концовок).

```jsonc
{
  "version": 1,
  "quests": [
    {
      "id": "story_tour",
      "name": "Тур",
      "order": 1,                  // порядок отображения
      "autostart": true,           // доступен сразу (без кнопки «Начать»); у остальных нет
      "wiki": "https://...",
      "description": "...",
      "rewards": ["...", ...],     // опционально
      "stub": "...",               // опционально: заглушка, если на вики ещё нет подзадач (тогда stages: [])
      "stages": [
        {
          "title": "Пролог",
          "objectives": [
            {
              "id": "tour-s1-1",   // уникальный id подзадачи (для хранения прогресса)
              "action": "Выход",   // человекочитаемое действие
              "desc": "Выбраться из Эпицентра",
              "maps": ["map_1"],   // опционально: привязка к локациям (для карты/«Найти в рейде»)
              "optional": true,    // опционально: не блокирует открытие следующей стадии
              "fir": true,         // опционально: найти в рейде
              "count": 3,          // опционально
              "handIn": [          // опционально: что добыть/сдать (для блока «Найти в рейде»)
                { "category": "Стройматериалы или Инструменты", "count": 5, "fir": true },
                { "name": "Аккумуляторная батарея", "count": 3 },  // name резолвится в реальную иконку, если есть в реестре
                { "item": "<itemId>", "count": 1 }
              ]
            }
          ]
        }
      ],
      // опционально: ветвление после общих стадий
      "branchPoint": "Выберите концовку:",
      "branches": [
        { "id": "savior", "label": "Концовка «Спаситель»", "stages": [ Stage, ... ] }
      ]
    }
  ]
}
```

**Логика стадий.** Стадия выполнена, когда отмечены все её **необязательные** цели — `optional`
не блокируют. Активна первая невыполненная стадия (все предыдущие выполнены); будущие скрыты.
Если есть `branches`, после общих стадий показывается выбор ветки; дальше — стадии выбранной ветки.
Прогресс хранится в IndexedDB: `storyDone` (id выполненных подзадач), `storyStarted` (начатые главы),
`storyBranch` (выбранная ветка по квесту).

## `keys.json` — справочник ключей по локациям

Генерируется `node build/keys.mjs`. Локация ключа берётся из **замков карт** (`Map.locks` API tarkov.dev) —
надёжный источник (у предмета-ключа прямого поля локации нет). Ключи без замка (автомобильные,
сейфовые и т.п.) идут в группу «Без привязки».

`wiki` — ссылка на **русскую** вики (`/ru/wiki/`), построенная по имени ключа (как для квестов).
`wikiLink` из API ведёт на английскую вики, поэтому не используется. Существование ru-страницы
проверяется через MediaWiki API; если страницы нет (~12 ключей, в основном вырезанный контент) —
ссылка ведёт на **поиск по ru-вики** (`Special:Search`), чтобы всё оставалось на русском.

```jsonc
{
  "meta":     { "source", "lang", "keyCount", "locatedCount", "note" },
  "mapOrder": ["Улицы Таркова", "Берег", ...],   // порядок групп (по числу ключей убыв.)
  "keys": [
    {
      "id", "name", "shortName", "icon", "img",
      "wiki",                     // русская вики (статья или Special:Search, если статьи нет)
      "maps": ["Таможня"],        // канонические карты (ночь/21+ слиты); [] = без привязки
      "lockTypes": ["door"]       // типы замков
    }
  ]
}
```

## `maps-geo.json` — гео-конфиг карт

Генерируется `node build/maps.mjs` из `maps.json` (the-hideout/tarkov-dev) + API + SVG.
Ключ — slug карты (`customs`, `interchange`, …).

```jsonc
{
  "customs": {
    "svg": "data/maps/customs.svg",     // локальная копия (скачивается build/maps.mjs в data/maps/)
    "svgRemote": "https://assets.tarkov.dev/maps/svg/Customs.svg", // фолбэк, если локальной нет
    "rotation": 0, "sx": ..., "sy": ..., "bounds": [[x1,z1],[x2,z2]],  // трансформация координат -> проценты
    "floors": ["Ground_Level", "Underground_Level", "First_Floor", "Second_Floor", "Third_Floor"], // слои SVG по порядку
    "base": ["First_Floor"],          // слои, показываемые вместе с землёй (без отдельной кнопки)
    "floorNames": { "Underground_Level": "Подвал", "Second_Floor": "2 этаж" }, // подписи кнопок-этажей
    "extents": { "<floor>": [[x1,z1,x2,z2,hmin,hmax], ...] },  // для привязки квест-маркеров к этажу
    "extracts": [ { "name", "faction", "x", "z" } ],          // только PMC/общие
    "bossSpawns": [ { "x", "z", "bosses": ["..."] } ],
    "bosses": [ { "name", "chance" } ]
  }
}
```
